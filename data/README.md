# Data Files Description

This document explains the purpose of the files stored under the `data/` directory.

## Directory Overview

```text
data/
|-- raw/
|-- processed/
|-- analysis/
`-- modeling/
```

- `raw/` stores downloaded source data. These files are raw inputs and can be regenerated by download scripts.
- `processed/` stores cleaned and feature-engineered datasets used by analysis and modeling scripts.
- `analysis/` stores statistical analysis outputs in CSV/JSON format.
- `modeling/` stores machine learning and deep learning model outputs.

## `data/raw/`

### `data/raw/jolpica/`

Raw JSON files downloaded from the Jolpica-F1 API.

Generated by:

```powershell
python download_jolpica_f1_data.py
```

Validated by:

```powershell
python validate_jolpica_f1_data.py
```

Main contents:

- `2019/` to `2026/`: season-level API files.
- `races.json`: race calendar and circuit information.
- `results.json`: race result records.
- `qualifying.json`: qualifying result records.
- `driverStandings.json`: driver standings.
- `constructorStandings.json`: constructor standings.
- `drivers.json`: driver metadata.
- `constructors.json`: constructor metadata.

These files are source data and are ignored by Git because they can be downloaded again.

## `data/processed/`

### `f1_model_dataset.csv`

Modern driver-race modeling table built from Jolpica-F1 raw JSON.

Each row represents one driver in one race.

Generated by:

```powershell
python build_f1_model_dataset.py
```

Used by:

- `build_f1_features.py`
- `build_f1_extended_features.py`

### `f1_features.csv`

Feature-engineered modern dataset based on `f1_model_dataset.csv`.

It adds pre-race standings, recent form, history counts, and data-quality flags.

Generated by:

```powershell
python build_f1_features.py
```

Used by:

- `analyze_f1_basic_stats.py`
- `visualize_f1_analysis.py`
- model scripts when the extended feature table is unavailable

### `f1_2026_schedule.csv`

Full 2026 race schedule with a flag showing whether each race already has result data.

Generated by:

```powershell
python build_f1_model_dataset.py
```

Used by:

- `analyze_f1_basic_stats.py`
- later 2026 prediction work

### `build_summary.json`

Metadata summary for `f1_model_dataset.csv` and `f1_2026_schedule.csv`.

It records generation time, output paths, fields, row counts, and 2026 completed rounds.

### `feature_summary.json`

Metadata summary for `f1_features.csv`.

It records feature fields, row counts by year, missing qualifying rows, and grid-zero rows.

### `f1_model_dataset_extended.csv`

Extended modeling table combining:

- SQLite historical records from 2003 to 2017
- Jolpica-F1 modern records from 2019 to 2026

Generated by:

```powershell
python build_f1_extended_features.py
```

This file increases the modeling sample size for machine learning experiments.

### `f1_features_extended.csv`

Extended feature-engineered table based on `f1_model_dataset_extended.csv`.

This is the preferred input for current machine learning and deep learning scripts.

Used by:

- `train_f1_podium_model.py`
- `train_f1_podium_deep_model.py`

### `extended_feature_summary.json`

Metadata summary for the extended feature dataset.

It records:

- historical source database
- modern source file
- historical year range
- total row count
- rows by year
- explanation for using 2003 as the historical starting point

## `data/analysis/`

This folder stores statistical analysis outputs.

Generated by:

```powershell
python analyze_f1_basic_stats.py
python analyze_f1_historical_sqlite.py
```

### Modern analysis files

These files mainly use complete modern seasons from 2019 to 2025. The 2026 season is handled separately as current-season state data.

- `dataset_overview.csv`: dataset size and entity counts.
- `dataset_overview_by_year.csv`: yearly row, race, driver, constructor, circuit, podium, and top-10 counts.
- `grid_finish_summary.csv`: correlation and rate summary for grid and race results.
- `grid_position_summary.csv`: outcome rates by starting grid position.
- `qualifying_position_summary.csv`: outcome rates by qualifying position.
- `feature_correlation_summary.csv`: Pearson and Spearman correlations between features and target outcomes.
- `driver_summary.csv`: driver-level total points, wins, podiums, top-10s, and average positions.
- `driver_points_by_year.csv`: driver performance by season.
- `constructor_summary.csv`: constructor-level total points, wins, podiums, top-10s, and average positions.
- `constructor_points_by_year.csv`: constructor performance by season.
- `constructor_competitiveness_by_year.csv`: constructor points concentration, HHI, and effective constructor count.
- `pre_race_strength_bins.csv`: podium/top-10 rates by pre-race strength bins.
- `circuit_summary.csv`: circuit-level pole win rate, front-3 podium rate, and position-change statistics.
- `circuit_grid_importance_score.csv`: composite circuit grid-importance score.
- `driver_position_gain_summary.csv`: driver grid-to-finish position gain summary.
- `constructor_position_gain_summary.csv`: constructor grid-to-finish position gain summary.
- `reliability_by_year.csv`: classified, incident, and mechanical/other status rates by year.
- `driver_reliability_summary.csv`: driver reliability summary.
- `constructor_reliability_summary.csv`: constructor reliability summary.
- `teammate_comparison.csv`: teammate comparison by season and constructor.
- `circuit_volatility_index.csv`: circuit volatility score based on position changes and surprise winners.
- `analysis_summary.json`: metadata summary for modern analysis outputs.

### 2026 current-season files

- `current_2026_driver_standings.csv`: current 2026 driver standings from completed races.
- `current_2026_constructor_standings.csv`: current 2026 constructor standings from completed races.
- `completed_2026_results.csv`: completed 2026 race results.
- `remaining_2026_schedule.csv`: remaining 2026 races without completed race results.

### Historical analysis files

These files are generated from `Formula1.sqlite` and are used as historical background.

- `dataset_overview_historical.csv`: historical database coverage and entity counts.
- `races_by_year_historical.csv`: number of races by year.
- `driver_wins_historical.csv`: top historical race-winning drivers.
- `constructor_wins_historical.csv`: top historical race-winning constructors.
- `grid_finish_summary_historical.csv`: historical grid-to-finish summary.
- `grid_position_summary_historical.csv`: historical outcome rates by grid position.
- `analysis_historical_summary.json`: metadata summary for historical analysis outputs.

## `data/modeling/`

This folder stores model evaluation outputs and prediction tables.

Generated by:

```powershell
python train_f1_podium_model.py
python train_f1_podium_deep_model.py
```

### Traditional machine learning outputs

- `podium_model_metrics.csv`: metrics for logistic regression, random forest, extra trees, calibrated random forest, and histogram gradient boosting.
- `podium_predictions_2025.csv`: predicted podium probability for each 2025 driver-race row.
- `podium_top3_predictions_2025.csv`: top 3 predicted podium candidates per 2025 race.
- `podium_feature_importance.csv`: feature importance for the selected tree-based model.
- `podium_logistic_feature_effects.csv`: logistic regression feature coefficients.
- `podium_rolling_backtest_metrics.csv`: expanding-window backtest results from 2022 to 2025.
- `podium_feature_mode_summary.csv`: average rolling-backtest performance by feature mode and model.
- `podium_completed_2026_probabilities.csv`: model probabilities for completed 2026 races.
- `podium_model_summary.json`: metadata summary for the traditional ML model pipeline.

### Deep learning outputs

- `deep_podium_model_metrics.csv`: MLP neural-network evaluation metrics.
- `deep_podium_predictions_2025.csv`: MLP predicted podium probabilities for 2025.
- `deep_podium_top3_predictions_2025.csv`: MLP top 3 predicted podium candidates per 2025 race.
- `deep_podium_training_history_post_qualifying.csv`: MLP training loss and validation score for post-qualifying features.
- `deep_podium_training_history_pre_race.csv`: MLP training loss and validation score for pre-race features.
- `deep_podium_model_summary.json`: metadata summary for the MLP model pipeline.

## Reproducibility Notes

The recommended data pipeline is:

```powershell
python download_formula1_dataset.py
python inspect_formula1_sqlite.py
python download_jolpica_f1_data.py
python validate_jolpica_f1_data.py
python build_f1_model_dataset.py
python build_f1_features.py
python build_f1_extended_features.py
python analyze_f1_basic_stats.py
python analyze_f1_historical_sqlite.py
python train_f1_podium_model.py
python train_f1_podium_deep_model.py
```

Raw downloaded data can be regenerated. Processed, analysis, and modeling files are generated outputs kept in the project for reporting and verification.
