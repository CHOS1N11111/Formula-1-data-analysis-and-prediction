# Formula 1 Data Analysis and Prediction

This project is a Formula 1 data engineering, analysis, and visualization project for a course assignment. It builds a reproducible data pipeline from raw datasets and API data to processed tables, statistical analysis, figures, and animated videos.

The current implementation focuses on data collection, validation, feature engineering, statistical analysis, and visualization. The processed data is also prepared for later podium and season-ranking prediction tasks.

## Project Scope

The project covers:

- Kaggle dataset downloading
- Jolpica-F1 API data downloading
- Raw data integrity validation
- SQLite database inspection
- Model-ready dataset construction
- Pre-race feature engineering
- Modern F1 analysis for 2019-2026
- Historical background analysis for 1950-2017/2018
- Static visualization for 2019-2025 and historical data
- Animated standings videos

For report figures, the visualizations intentionally exclude 2026 data. The 2026 data is kept for current-season state analysis and later prediction work.

## Setup

Install Python dependencies:

```powershell
pip install -r requirements.txt
```

For MP4 video generation, install `ffmpeg` and make sure it is available in `PATH`.

Check:

```powershell
ffmpeg -version
```

## Recommended Run Order

Run commands from the project root.

```powershell
python download_formula1_dataset.py
python inspect_formula1_sqlite.py
python download_jolpica_f1_data.py
python validate_jolpica_f1_data.py
python build_f1_model_dataset.py
python build_f1_features.py
python analyze_f1_basic_stats.py
python analyze_f1_historical_sqlite.py
python visualize_f1_analysis.py
python animate_f1_points.py
```

If the Kaggle datasets or raw Jolpica-F1 data already exist locally, the download steps can be skipped.

## Notes

- Static report figures use 2019-2025 modern historical data and historical SQLite data.
- 2026 data is not used in static report figures.
- 2026 data is retained for current-season status and later prediction.
- Raw data and videos are ignored by Git to avoid committing large or regenerated files.
